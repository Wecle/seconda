# Agent 面试房间 UX 验证清单

## 核心流式验收

1. 提交回答后，候选人消息先出现，随后才显示推理过程。
2. 推理过程自动展开，并随公开推理事件持续增长。
3. 推理期间手动折叠后，后续推理增量不得再次强制展开。
4. 收到 `response_started` 时，推理过程自动折叠。
5. 问题或结束语在数据库正式提交前已开始流式增长。
6. 重新展开推理过程后，可以看到修正路线和标为“已调整方案”的 discarded attempts。
7. 刷新或断线恢复后，从持久化 cursor 继续，不得重复推理或问题文本。
8. 终结完成只创建一个评分任务。

## 补充可靠性验收

1. 用户消息提交失败时保留失败消息与重试入口，不提前显示面试官处理状态。
2. 强制触发 Run 失败，确认 provisional 问题消失、失败面板保持展开，终态 SSE 不再自动重连。
3. 断开网络，确认最多自动重连五次，之后只显示手动重连入口。
4. 结束面试，确认依次显示评分进度和报告生成状态，只有 `completed` 后才能打开报告。
5. 在运行、评分、报告、失败状态刷新页面，确认正式消息、状态区块和进度均从 PostgreSQL 恢复。
6. 检查 Network：初始详情 GET 无并发重复，终态后无持续 Run/SSE 请求。
7. 确认工具生命周期只展示 `publicLabel`，不展示工具参数、结果、Prompt 或数据库标识。
